Currently we have a consensus to not add parcels in OSM. What you can do is use them as external resource like aerial images when it can help tagging on-the-ground features e.g. walls, fences, landuses changes limits, etc. But when land boundary defining land owner are purely virtual, it would create unncessary complexity on the map edition for the future contributors and will be hard to maintain (constantly change by merges and splits).

You can find a more developed list of arguments about parcels in OSM on this wiki page:

http://wiki.openstreetmap.org/wiki/Parcel
