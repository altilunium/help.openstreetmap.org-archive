You can extract data, edit it, and then use it for navigation devices such as those from Garmin. It isn't, as far as I am aware possible to create something using uMap (which creates a web map) and then from that extract data in a format that e.g. a satnav can understand. If you could provide a bit more detail about what you're trying to do it would be easier to reply.

It is possible to create a custom map using OSM tools, but please don't upload it to OSM unless it reflects the reality on the ground, and don't add tags that are useful for "my satnav" but not for anyone else.

For licensing questions, follow the links from the "copyright" page.

To start with the last part of your question: yes, you can use OSM dereived data commercially, there may be some restrictions on how and certain obligations on your behalf depending on the specifics.

See http://www.openstreetmap.org/copyright, http://wiki.osmfoundation.org/wiki/License/Community_Guidelines and http://wiki.osmfoundation.org/wiki/License

Back to the start, in general POI data, speed limits, street names etc. should be recorded in the OSM data set, "custom maps" should be produced by taking the OSM data and filtering for what you actually want to use. Without specifics about the target navigation device it is difficult to be more specific than this.
