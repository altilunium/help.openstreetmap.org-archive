There is an input plugin in mapnik that can directly read .osm files. However, you will have to develop your own stylesheet for it, or heavily adapt the default osm.xml stylesheet.

Even if you get it running that way, it will still not support everything the default map will show. Some things are done in the SQL queries, and you'll loose that in the osm plugin. Also, I read that the osm plugin doesn't quite handle relations as well.
