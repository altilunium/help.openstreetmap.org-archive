I'm going through the roads in my area and tagging the surfaces as paved/unpaved. Many roads are paved for some portion and then unpaved for another portion. So far I have been splitting the way in two and giving each new way the appropriate surface= tag. When I split the way for a road into parts, should I be creating a new relation that contains the ways?

If so, I'm guessing that the relation would be type=route and route=road with the name being whatever the road name is. Is this correct?

Most people would only create a "route" relation for long stretches of road, e.g. a motorway or long distance route, not for inner-city streets. It is common for inner-city streets to consist of a number of individual ways that have the same name, and no relation.
