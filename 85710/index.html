Italian

Prima di segnalare un utente dovresti cercare risolvere il problema contattandolo oppure commentando un suo changeset; lo hai fatto?

Se non si riesce in questo modo, a mio avviso dovresti provare a contattare la comunità locale o quella italiana usando la mailing list talk-it, il gruppo Telegram @OpenStreetMapItalia oppure il nuovissimo gruppo Discourse della comunità italiana

Quando fai delle modifiche, cerca di spiegare cosa hai fatto nel commento del changeset e di specificare le fonti che hai usato; ciò aiuterà gli altri a capire come comportarsi con i dati da te inseriti o modificati.

English Translation

Before reporting another user you should try solving the issue by contacting them or commenting the changeset of contention; did you do that?

If that fails, my advice is to contact the local community or the Italian community using the mailinglist talk-it, the Telegram group @OpenStreetMapItalia or the Italian community brand new Discourse group

When you edit the map, try to explain what you just did in the changeset comment and which sources you have used; this will help other users understanding what to do with the data you have created or modified.

A solution might be to copy the current boundary that is in osm to a new layer in josm. Revert the changeset to the previous version you think is correct. Then merge the boundary you saved in the new layer in to osm and tag it as a proposed park. The proposed park won’t render in osm but you can add a note tag to each with an explanation. Then when the changes to the park boundaries have been decided you can make necessary changes and make the proposed boundary the actual new boundary. You may prefer to have the new one taggged from proposed to construction for a period until implementation is complete. This may be an agreed compromise way forward.
