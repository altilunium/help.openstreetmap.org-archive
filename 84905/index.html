OSM's Data Working Group** get lots of complaints about incorrect place names in Windy's apps and websites such as windy.com. I believe that in all of the place name reports that we've had so far, Windy has been at fault and not OSM.

In this case, Windy's website actually has the correct name so hopefully their app will update when they get around to it.

Why Windy were using the name Nicomedia I've no idea - you'd have to ask them that. The Izmit node has had the same language names for a couple of years at least. Nicomedia is in OSM as the latin name for Izmit, which is correct, but there's no evidence that Windy got their incorrect name from there. A number of Windy's other incorrect names don't have an obvious source in OSM either (one example was the same city name duplicated 30km to the east).

** I'm a member of that group
