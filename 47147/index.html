This question has been asked before, albeit not as considerately as this time.

The basic answer remains the same: this is an artefact of the label collision algorithm of Mapnik. Philadelphia/Camden is just one of many examples.

There are OSM-based maps which do much more pre-processing to support label placement: one example is mapsurfer.net (see Philadelphia here, and note that Camden does not appear until well zoomed in). The standard OSM map is built for speedy update and therefore tends to eschew pre-processing of the data.
