OSM does not have good address coverage in all areas. If addresses are missing in the OSM database then they will not show up in OsmAnd offline files. Here in the U.S. there is some address data available from the Census Bureau which is automatically incorporated into the Nominatim data set. So if you tap the "Internet" button on the OsmAnd address search page and do a Nominatim query, it will use that. But if you are not in the U.S. or if you are wanting to use it offline then this won't help you.

But by all means, please do add address data to OSM! One pretty quick way of doing this is to use KeypadMapper on your Android phone. You can just walk along a street recording address numbers as you go. I collected most of the addresses in this area on a single walk:
http://www.openstreetmap.org/?lat=39.21141&lon=-96.62054&zoom=17&layers=M

Sometimes addresses can be imported from other (government) sources but this is an advanced operation that you should not attempt until you are a well practiced mapper and have consulted the rest of the community.

You can map multiple address nodes as part of a single building. For example in my apartment complex, there are two stairwells, each with their own address, per building. I put the address information on nodes that are a part of the building outline way with an entrance=stairwell tag. Example node:

http://www.openstreetmap.org/browse/node/802691705
