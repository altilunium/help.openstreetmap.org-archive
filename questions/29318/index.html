www.openstreetmap.org doesn't has the primary goal to be a replacement to enduser portals as Google Maps for example. Currently it's more a webmap for the contributors to see the rendered geodata and explore the areas. This is for example the reason, why the default mapstyle has some visual clutter as it contains a lot of objects and attributes (more that is needed for a single dedicated user scenario like car driving). The role of the portal is discussed controversal, some aspects can be found here: http://osm.gryph.de/2012/02/google-is-not-the-enemy/

But the advantage of OSM is that there is a big commmunity that creates mashups and a lot of tools to work with the data and the rendered maps, that aren't visible at the main website:

  • http://wiki.openstreetmap.org/wiki/Maps
  • http://wiki.openstreetmap.org/wiki/Frameworks

For your specific request, you might use this route editor: http://www.pifpafpuf.de/cycleroute/map

You can't do this on the OpenStreetMap.org Front page, but there's various other places on the web where you can overlay KML on top of OpenStreetMap easily.

  • 'Web Maps Lite' KML displayer (my own crappy old experiment)
  • Example Openlayers KML display
  • UMap lets you create a map and then 'import' a KML file (up arrow icon on the right)

There's probably others. These are also listed at KML#Simple KML viewing on the wiki. ...mind you I haven't tested these with a KMZ file.
