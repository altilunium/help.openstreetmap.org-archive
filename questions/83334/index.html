guard_rail looks to be mainly used to say whether a guard_rail exists or not along a given highway segment. It would probably be better to use a different tag, such as guard_rail:type. Note that Jersey Barriers have had a distinct tag for a long time.

(07 Feb '22, 11:31) SK53 ♦

@SK53 Both are possible. Eg: kerb= on a a feature for what kerb has; and on a barrier=kerb for what it is. guard_rail:type= is not meaningful enough, especially when it may be mistakened as a double-sided guard rail (guard_rail=double is only a generic solution for now).
