I assume you mean streets when you write "routes", then the answer is yes. The data that is used to create the map (the image you see on the website) is coming from another database than the data that is used to calculate the route you can follow from A to B. That second database is updated less frequently and hence, the calculated routes do not necessarily use the streets that are visible on the map. I believe that the same problem also occurs on the website of osrm itself. The osm.org just integrates services from multiple websites and this causes this "weird" phenomenon.

Yes. Rendering and routing are different processes. They need different databases. In this case, these services don't even run on the same servers. Moreover, rendering is done my OSM itself while currently all routing services are more or less provided by third-parties.
