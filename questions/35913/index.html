Wisepilot isn't part of OpenStreetMap; it's a separate software application that just happens to use OSM data. If you want software support for it you'll have to ask them.

Incidentally, their website now has an OpenStreetMap page which contains a "How to become an editor" section. Errors reported in WisePilot do create anonymous notes in OpenStreetMap, so do take care when reporting issues that they're reported in the right place and apply to current OpenStreetMap data (rather than what might have loaded in the app). This mailing list thread has some more information about the "notes" issue.

To get Navteq updates for Wisepilot you have to ask the manufacturer of Wisepilot. Write them, call them if you cannot find anything in their FAQ/Support site.

Adding OSM-based maps yourself to Wisepilot would only be possible for yourself if the used map format is known and not secret. I could not find info on our wiki page about this.

In case you did not know: OSM is just about the raw data – not really about finished maps. Likely the maps which are used with Wisepilot are made by the Wisepilot manufacturer – based on the OSM data.
