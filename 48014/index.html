One possibility is that the API has a short hiccup and is therefore refusing the connection. If you see uploads succeeding this is less likely.

You should be able to enter your OSM username and password just as you enter it on the osm homepage. Try to log in there to verify that your credentials are working.

If that works but you cannot get JOSM authentication to work (whyever?!) ... you could try to set up OAuth instead of username+password in JOSM. Go to: JOSM's Preferences -> Connection settings, OAuth (help).

Temporary workaround: You could save your edits into a .osm file and upload it later.
