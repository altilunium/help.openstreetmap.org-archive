The rendering rules for the "standard" style are located here https://github.com/gravitystorm/openstreetmap-carto

The relevant rules for rendering names are in https://github.com/gravitystorm/openstreetmap-carto/blob/master/addressing.mss

As you can see, if you tagged the name with the "name" tag, when it gets rendered depends on the zoom level and the size of the polygon, not on orientation.

If you need to render the names earlier I would suggest using tilemill and creating a custom version of the style.

@yoonghm: Simon mentions «"standard" style» – there are many other styles. And someone caring especially for building names could make his own style (for our OSM data). You can find a small selection of other styles by using the layers function of the osm.org homepage. All styles may change/improve in future, so please care that the data is correct, not that the standard style shows it in the way you would like to see it. And if you have a improvement suggestion you can contact the style programmers at the github page (new account needed).
