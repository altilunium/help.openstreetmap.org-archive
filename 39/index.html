There are two localization levels:

  1. to have just UI strings translated, this is technical task, and possible with OSQA (see http://meta.osqa.net/questions/94/how-can-i-compilecreate-a-new-language-pack) for instance.
  2. to have whole Q&A content in German - either similar general questions like in English, or even German-specific questions. This is a bit more complex task technically, possibly requires another separate instance of the help system (help.osm.de?). But is it really good idea, it would generate similar questions duplicated ?
