My answer is similar to jgpacker's, but can be done more quickly and all within JOSM.

  • Download the area containing all of the buildings in JOSM
  • Search (Ctrl+F) for building_number. This will select all objects in the area you downloaded that have this tag
  • Scan down the list of selected objects to make sure it contains only the things you want to change, namely the university buildings you tagged with building_number
  • In the tag box at the top right, it will have a line for the tag building_number with the value < different >
  • Click edit for this tag and change the key field from building_number to whatever new key you want to use, like ref, but don't touch the value field, leaving it as < different >

What this will do is retain the respective value field for each of the objects, but change the key for all of them. For example, building_number=100 would become ref=100. This saves having to edit each one individually or edit an .OSM file.

In JOSM, you can simply search(ctrl+f) for building_number or building_number:* and you will find them. That's probably the first thing you want to do to verify that only the buildings you added that actually have this tag.

If you want to automate this process, you may do the following:

After downloading the university via JOSM, save it as a .osm file. Close JOSM and open the .osm file in a text editor, and use the text editor's "replace all" functionality on building_number to the tag you desire to use. Then open this file in JOSM, download the data from OpenStreetMap, and merge the layers of the file with the OSM Data Layer. Send Changes.

In this case, you probably want to use ref=* or ref:unimelb=* instead of unimelb:building_number=*.
